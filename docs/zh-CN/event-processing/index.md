# 事件处理

`ingest4x` 的事件处理以 WAL 重放阶段的单个 Rhai 入口为中心：

| 作用 | 入口函数 | 阶段 | 责任 |
| --- | --- | --- | --- |
| [转换与投递](event-transformation.md) | `fn process(event, request)` | WAL 重放 | 在脚本内联完成字段校验、变更/扩展字段并决定投递目标 |

上述两阶段都不在 ingress `/ingest/{project_key}` 请求线程执行。`/ingest/{project_key}` 仅处理项目解析、鉴权、体积校验和写 WAL；真实处理发生在后台重放中。

## 处理流程

```text
+--------------------------------------------------------------------------------+
| Replay worker                                                                  |
|                                                                                |
| +------------+    +-----------------------------------------------+            |
| | WAL record | -> | 带内联校验 helper 的 Rhai processor          |            |
| +------------+    +-----------------------------------------------+            |
|                                                |                               |
|                                                v                               |
|                                      +----------------------+                  |
|                                      | Delivery-ready event  |                  |
|                                      +----------------------+                  |
+--------------------------------------------------------------------------------+
```

每条 WAL 记录在重放时会加载当前项目绑定的处理脚本。若无自定义处理脚本，则使用默认脚本。
