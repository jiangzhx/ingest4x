# 事件处理

`ingest4x` 的事件处理分为两阶段，均由 Rhai 脚本实现：

| 作用 | 入口函数 | 阶段 | 责任 |
| --- | --- | --- | --- |
| [校验](event-validation.md) | `fn validate(event)` | WAL 重放 | 校验事件字段是否满足项目规则 |
| [转换与投递](event-transformation.md) | `fn process(event, request)` | WAL 重放 | 执行规则校验、变更/扩展字段并决定投递目标 |

上述两阶段都不在 ingress `/ingest` 请求线程执行。`/ingest` 仅处理鉴权/体积/写 WAL；真实处理发生在后台重放中。

## 处理流程

```text
+--------------------------------------------------------------------------------+
| Replay worker                                                                  |
|                                                                                |
| +------------+    +--------------------+    +--------------------------+        |
| | WAL record | -> | Validation rules    | -> | Transformation processor  |        |
| +------------+    +--------------------+    +--------------------------+        |
|                                                |                               |
|                                                v                               |
|                                      +----------------------+                  |
|                                      | Delivery-ready event  |                  |
|                                      +----------------------+                  |
+--------------------------------------------------------------------------------+
```

每条 WAL 记录在重放时，先加载当前项目绑定的校验规则，再加载该项目处理脚本。若无自定义处理脚本，则使用默认脚本。

## 文档

- [校验](event-validation.md)：字段路径、存在性/类型约束、结果语义与规则集存储模型。
- [转换与投递](event-transformation.md)：`process(event, request)`、`validate(event)`、`emit(...)`、sink 常量、请求上下文、模块与绑定。
